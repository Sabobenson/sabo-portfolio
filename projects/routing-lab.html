Routing Lab Project

Interconnecting Two Networks Using a Router

1. Introduction

Modern networks require communication between multiple isolated subnets. This project demonstrates how a router is configured to enable communication between two separate IP networks using basic Layer 3 routing principles.

2. Objective

  • To design a simple multi-network topology
  • To configure router interfaces with correct IP addressing
  • To verify connectivity between two different networks

3. Network Topology

The network consists of two LANs connected through a router. Each LAN contains a PC and a switch, forming two distinct subnets.

4. IP Addressing Scheme

Device Interface IP Address
PC1 NIC 192.168.1.2 /24
Router G0/0 192.168.1.1 /24
Router G0/1 192.168.2.1 /24
PC2 NIC 192.168.2.2 /24

5. Configuration

Router Configuration

enable
configure terminal

interface g0/0
ip address 192.168.1.1 255.255.255.0
no shutdown

interface g0/1
ip address 192.168.2.1 255.255.255.0
no shutdown

end

PC Configuration

PC1

  • IP: 192.168.1.2
  • Gateway: 192.168.1.1

PC2

  • IP: 192.168.2.2
  • Gateway: 192.168.2.1

6. Testing & Verification

Connectivity between the two networks was verified using ICMP (ping). A successful response confirmed that routing was correctly configured.

7. Challenges Encountered

  • Forgetting to enable router interfaces (no shutdown)
  • Incorrect default gateway configuration
  • IP addressing mistakes

8. Conclusion

The experiment successfully demonstrated how routers facilitate communication between different IP networks. Proper configuration of interfaces and addressing is essential for network connectivity.

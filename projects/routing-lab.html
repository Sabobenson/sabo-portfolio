Small Office Network Routing Setup

By Sabo Benson – Network Solutions Focused Engineer

1. Problem Statement

In a small office environment, two departments were operating on separate networks. Devices in these networks could not communicate due to lack of routing, leading to inefficiency and poor resource sharing.

2. Solution

A router was configured to enable communication between the two networks. Proper IP addressing and interface configuration were implemented to ensure reliable connectivity.

  • Designed network topology
  • Configured router interfaces
  • Assigned IP addresses
  • Verified connectivity

3. Network Topology

PC1 → Switch1 → Router → Switch2 → PC2

Add your topology image here (assets/images/topology.png)

4. IP Addressing Scheme

  • PC1: 192.168.1.2 /24
  • Router G0/0: 192.168.1.1
  • Router G0/1: 192.168.2.1
  • PC2: 192.168.2.2 /24

5. Configuration

enable configure terminal interface g0/0 ip address 192.168.1.1 255.255.255.0 no shutdown interface g0/1 ip address 192.168.2.1 255.255.255.0 no shutdown end

6. Testing & Verification

Ping test from PC1 to PC2:

ping 192.168.2.2

Result: Successful replies received. This confirms that routing between the two networks is functioning correctly.
